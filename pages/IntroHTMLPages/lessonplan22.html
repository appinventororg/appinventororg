Lesson Plan 2.2: MoleMash Tutorial, Conceptualize, Customize

Google Doc

Objectives

  • Learn about image sprites and moving them around on a Canvas.
  • Learn about animation through the Clock.Timer event
  • Learn how to generate a random number.
  • Learn how to increment a value
  • Introduction to procedures

Outline

1. Mini-tasks: Playing a sound over and over, moving a spaceship.

2. Students build “MoleMash”

3. MoleMash: Conceptualize

4. MoleMash: Customize

Preparation

Print out copies of the Conceptualize and Customize worksheet.

1. Warmup Task(30)

Introduce the Clock component, its TimerInterval property, and its Clock.Timer event handler.

The following tasks can be assigned as programming tasks using App Inventor, or you can ask the students to sketch the blocks on paper.

  • Assign students the task of playing a sound every five seconds. You can provide them with a sound clip, which should be a short one. After a few minutes, demonstrate the solution: http://appinventor.org/TimedActivity
  • Assign the students the task of making an image sprite (e.g., spaceship) move smoothly across the screen. After a few minutes, demonstrate the solution: appinventor.org/TimedActivity#spaceship

Emphasize that any blocks you put in Clock.Timer will be executed over and over, every Clock.TimerInterval milliseconds. Be sure and explain that a millisecond is 1/1000 second.

Emphasize also that in App Inventor time is treated as just another event. Ask them to name all the events they have dealt with. Thus far they have been user interface events-- button clicks and Canvas.Dragged events. But mention external events like Texting.Message Received and LocationSensor.LocationChanged (for when a GPS reading comes in), and of course timer events.

2. Students build “AndroidMash”(45)

Ask the students to build MoleMash using the book chapter tutorial: http://www.appinventor.org/MoleMash2 or the AndroidMash using the video http://www.appinventor.org/AndroidMash-steps. The tutorials are essentially the same, but you should go over the one(s) you use carefully so you’ll be ready for student questions.

Optionally, ask the students to create a portfolio page and post the MoleMash app on it. You may want to do this to ensure students complete these tasks and/or for the students to have a record of their work. But you might also want to only require “publishing” for creative projects.

3. Discussion (45)

Conceptualize and Customize Worksheet:

https://docs.google.com/document/d/1IgKiNpPMWlspEwi5LRehjBHAQaFR_aUHsJNhfeuTT2c/edit#

The discussion questions discuss a number of topics: canvases, moving sprites on a canvas, incrementing values, and procedures.

Incrementing values, x=x+1, is a difficult concept and you should definitely spend time demonstrating how a memory cell is accessed and then changed with this operation, and clearly defining set and get operations.

You have some leeway in terms of how deep you want to go into procedures at this point. The next lessons do not depend on procedures, so one option is to just briefly introduce them and move on. Other instructors prefer to emphasize procedures early on, however.

4. Customization Exercise (30-90)

The first customization exercise is really a creative task-- the students personalizing the Mole Mash game with their own media and sound effects. Though this task doesn’t teach new programming skills, students are happy to spend ample time working on it, and it can really be motivating. Remember, most have never been given the capability of modifying software, so being able to create a custom game is pretty cool.

Another customization task involves speed, and is a good way to introduce some math and simple physics, e.g., speed = distance/time.

Another customization task involves introducing an image the player doesn’t want to hit. For this task, help the students to understand that each time the player touches a sprite, two events are triggered: 1) Canvas.Touched, and 2) ImageSprite.Touched. In the tutorial, Canvas.Touched is used only. But if you want to distinguish between which ImageSprite is hit you must program each of their Touched event handlers.
