Algorithms

An algorithm is a step-by-step set of operations to be performed to solve a problem.

An algorithm is like a recipe, but more precise.

A procedure is a named set of blocks, a realization of an algorithm for a specific computer and programming language.

MoveRandom provides a name for some complicated blocks. Once you create the procedure, you have a new block, moveRandom, which you can call as in the example.

Using procedures is key to programming and how programmers use abstraction to break down problems. Consider this from Ellen Spertus in chapter 3 of the App Inventor 2 book.

The ability to name and later call a set of instructions like moveRandom is one of the key tools in computer science and is referred to as procedural abstraction. It is called “abstraction” because the caller of the procedure (who, in real-world projects, is likely to be di‚fferent from the author of the procedure) only needs to know what the procedure does (moves the mole), not how it does it (by making two calls to the random number generator). Without procedural abstraction, big computer programs would not be possible, because they contain too much code for individuals to hold in their head at a time. This is analogous to the division of labor in the real world, where, for example, di‚fferent engineers design di‚erent parts of a car, none of them understanding all of the details, and the driver only has to understand the interface (e.g., pressing the brake pedal to stop the car), not the implementation.

Essentially, a procedure lets you name some detailed instructions then use that name (that block) elsewhere in your program. The “abstraction” lets programmers focus on the overall problem at different levels of detail. Here are just some of the advantages of using procedures:

• If you need to fix a bug or change something, it only needs to be changed in one place.

• Procedures can be collected into a library and used in di‚fferent programs.

• Breaking code into pieces helps you “divide and conquer” a problem.

• It is easier to test code if it is neatly segregated from the rest of the program.

• Procedures make it easier for someone else (or you, a month later) to read the code.

Logo

Logo is a classic drawing app that was one of the first programming environments for kids. You can draw using some simple primitive blocks. In this first Android Logo version, the following procedures (blocks) have been defined for you:

   forward  -- moves the android ahead 10 pixels

   turn -- rotates the android 90 degrees
  penup -- if the pen is up you can move android but not draw

   pendown -- if the pen is down drawing happens on forward.
