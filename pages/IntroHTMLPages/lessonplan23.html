Lesson Plan 2.3: Animated Movement

Google Doc

Objectives

  • Understand how animation works (with both clock timers and ImageSprite properties)
  • Learn about speed, frames-per-second and and direction in animation movement.
  • Gain experience using variables and if-else.

Outline

1. Image Sprite Introduction

2. Animation Challenge

3. Pong Tutorial

4. Creative Game Project

1. Image Sprite Introduction (15)

Demonstrate how to animate sprites using their internal properties (heading, speed, interval). Emphasize that the internal sprite properties are an alternative to using a Clock component and Clock.Timer (Clock.Timer can be used to trigger any actions, sprite animation is just above moving objects).

Assign the sprite animation nugget: http://appinventor.org/sprites and/or demonstrate topics in class.

2. Animation Challenge (60)

You can print out the animation challenge here: https://docs.google.com/document/d/1U7K6w02tIugKujQbH2RIyxMQCFuEgIWtg3Zlu2d-Gu0

The challenge prepares students for building animated games like Pong or Asteroid-like arcade games.

App Inventor provides two ways to program the animated movement of objects. The first way is that you can use a Clock and the Clock.Timer event to program any type of animation (movement, rotation, etc.). The second way is that the ImageSprite and Ball components have properties that allow you to program animated movement. These two methods should not be intermixed, but the students should be exposed to both, one reason being so they don’t intermix them (I’ve tried only showing one way or another, and had problems when students stumble upon the other one and end up doing both).

The animation challenge starts with a fairly simple task of moving an object in various directions (horizontal, vertical, and diagonal). It asks the students to use both schemes (Clock.Timer and sprite animation) to complete the task.

The challenge then has a discussion section about speed and frames-per-second. Its a nice way to introduce some Math and simple Physics.

The second coding challenge asks the students to make a ball go back and forth. This is pretty difficult and helps students understand the need for variables and if-else blocks.

3. Students develop the Pong game using tutorial (45)

This tutorial is currently only in video form: http://www.appinventor.org/pong-steps.

Alternative: LadyBug Chase is another game involving sprite animation and even orientation sensing for tilting the screen and moving objects.

http://www.appinventor.org/ladybug-chase-steps

4. Creative Game Project (60-120)

After building Mole Mash and Pong, the students should be ready to create any arcade-like game. Games have endless possibilities in terms of scoring, levels, and rules, so even if students don’t stray far from the samples, they can build something interesting and personal.

As an instructor, be clear on expectations in terms of what coding must be demonstrated, e.g., the game must have smooth animation, change of speeds, collisions, scoring,randomness, etc.
