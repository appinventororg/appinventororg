Animation Challenge:
Animate With Internal Components

As you probably noticed, animation with the Clock Component can become challenging, frustrating, and limiting.

For Example, it would be pretty difficult to get the ball to move in a circle using the Clock Component.
The people at MIT noticed this so they created internal properties that can animate balls and sprites on a Canvas without the Clock Component. Let take a look at the properties to see the difference between using the clock component to animate and the internal properties to animate.

Tasks

When the go button is clicked, do the following by using a ball's internal properties. (You should not be using the Clock to move the balls)

  1. Have one ball move right. When it hits right the edge, the ball should move left. When it hits the left edge, the ball should move right. This should happen continuously.
  2. Have another ball move up. When it hits the top edge the ball should move down. When it hits the bottom edge, the ball should move up. This should happen continuously.
  3. Have another ball move down-right diagonally. When it hits the edge, the ball should move diagonally up-left. When it hits the edge, the ball should move diagonally down-left. This should happen continuously.

Start by doing a SaveAs of your Coding Challenge 1 Project and call the new project "Coding Challenge 1 Sprites." Then when your new project opens delete the Clock component-- you won’t use it. Your goal is, once again, to have the three balls move smoothly across the canvas (one down, one left, and one diagonally). Try this challenge on your own first and then use this page for help.
