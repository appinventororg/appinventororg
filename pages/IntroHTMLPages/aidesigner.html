A Tour of the App Inventor Designer

About the Designer

When you create a new project in App Inventor, you begin in the Designer view. In the Designer, you construct how the app will appear and also add components that will be needed for the app’s functionality.

The Designer consists of four panels: the components Palette, the Viewer, the Components panel, and the Properties panel.

Components Palette


The component Palette on the left displays all of the components available in App Inventor-- all the User Interface widgets and tools you can use to make your app work.

The components are organized into folders.

The first folder, User Interface, contains items that the user can visually see at a basic level.
Ex. Button, TextBox, CheckBox, Image

The next folder, Layout, contains the different arrangements one can use to allow components to be next to each other.
Ex. HorizontalArrangement, VerticalArrangement, TableArrangement

The Media folder mainly contains items that use external tools a device has.
Ex. Sound, Camera, TextToSpeech

The Drawing and Animation folder contains the components to perform animations and drawings
through using the Canvas component.

The Sensors folder contains components that use sensors to trigger events,
Ex. LocationSensor, AccelerometerSensor, Clock

The Social folder contains components that involve communication.
Ex. Texing, PhoneCall, Twitter

The Storage folder contains components that can help your apps store user-generated information.
Ex. Tinydb, TinyWebDB, FusionTablesControl

The Connectivity folder contains components that can link to the web bluetooth or other apps within the device.

The LEGO® MINDSTORMS® folder contains components that can be used to control LEGO® MINDSTORMS® devices.

Viewer

To add a component to your app, drag the component into the white space in the Viewer (the large middle area). When the object’s been dragged, it does not get placed in the exact position you may want. Instead, it appears at the top-left of the screen. When another component is dragged in the Viewer, it will appear below the first component (not next to it).

Components panel

The component name will appear in the Components panel to the right of the Viewer. This panel allows you to rename and delete components.

Renaming is important because when you code an app's behavior in the blocks editor, you need to be able to distinguish each of your components by name.
When renaming your components, give them descriptive names or keep the component type as the suffix of the name.
Ex: if you have a red Button, rename it from its default (ex, Button1) to "RedButton."

Properties panel

The Properties panel is where you can change the settings for your components such as changing the color or text of a Button. The panel shows the properties for the currently selected components. For example, in this image, "Screen1" is highlighted, so the properties for "Screen 1" are displayed. Within this panel, you are also able to change the components alignment. Ex: If you wanted the Screen to align all of its components in the center, you would set the "AlignHorizontal" property to "Center."
