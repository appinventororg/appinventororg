Task4 - Make the ball bounce on the paddle

Now that we have the ball bouncing on the walls, we want to have the ball bounce when it hits the paddle.

Luckily there is an easy event for this known as the CollidedWith event which is triggered when a sprites collides with (or touches) any other sprite.

What do you think the event parameter “other” stands for?

Other tells the app which sprite it collided with.

Answer

What does this mean?

It means that we can say, for example, If other is a "coin" then gain a point, if other is a "monster" then lose a point. In this app there is only one sprite that we can hit, so we do not need to check which sprite is hit because there is only one.

Answer

Instead of using the Bounce block how could we get the ball to bounce up?

Heading controls the direction of the ball.
So we can change the heading each time the ball collides with the paddle.
But what should the heading be set to?

Math

If the ball goes straight down (current heading = 270) it should bounce straight back up (bounce heading = 90).
If the ball hits the paddle at a perfect 45 degree angle downward to the right (current heading = 315) it should bounce back up in another 45 degree angle (bounce heading = 45)

Algebra

Solve for x
x-270=90
x-315=45

270+90 = 360; x = 360
135+45 = 360; x =360
We are trying to find the formula for bouncing, or a pattern that occurs when bouncing. We can use our current heading to calculate the new heading after the ball bounces.
The formula is 360 - current heading = final heading
When Ball1 Collides (with the paddle) set its heading to 360 -current heading.
