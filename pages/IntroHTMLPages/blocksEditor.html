The Blocks Editor

The Blocks Editor is where you code the functional behavior of your app, where you specify how the app should respond to events. This section provides an overview of the Blocks Editor.

An App’s Behavior consists of Event-Handlers

For the most part, you’ll code the app’s behavior by defining event-handler blocks: blocks that specify how the app will respond to a particular event. For example, if you want the app to do something when the user clicks a button named “Button1,” you’ll first drag in a “when Button1.Click” block, then drag in response blocks within it. Changing Component Properties with Blocks

If you want to change the initial value of a component property, you do that in the Designer. You can also change property values dynamically-- as the app runs-- using blocks. Property set blocks are dark green with open slots to the right.

Set blocks allow you to change the value of a property.

You can also check the value of a property with a light green block. Though “get” doesn’t appear on the block, these are called get blocks because they get, or retrieve, the value of a property.

Function Calls

Function Calls are purple blocks that cause the app to do something, e.g., play a sound. Some function blocks, but not all, have opens lots at the end.

Here are the Function Calls for the Sound Component:

With these blocks, an app can play, pause, resume, or stop a sound clip, or it can cause the device to vibrate.

The Vibrate function block has a slot with millisecs next to it. This slot is called a parameter. A parameter provides information a function needs to do its job. In this case, you (the programmer) needs to tell the function call how long it should vibrate.

Built-in Blocks

In the Blocks Editor, there are drawers for “Built-in” function blocks that are not specific to a particular component. These blocks do all sorts of things including adding numbers and opening screens.

Built-in Blocks: Math

In the Math folder, you’ll see blocks for representing a number, for adding, subtracting, multiplying, and dividing, for getting the square root, sin, cos, tan, and other operations. There are even blocks for generating random numbers like you need in a game.

Built-in Blocks: Text

The Text folder provides blocks for manipulating text values, e.g., “cat.” There is a block to represent a text value (the empty quotes can be filled in), to combine texts, to get the length of text, and to compare texts. There are also advanced blocks to help with parsing text and splitting text into segments.

Built-in Blocks: Control

The Coltrol folder provides blocks for controlling the flow of how your blocks are executed. For instance, if you only want to execute some blocks under certain conditions, you’ll use an if block. If you want to continuously repeat blocks, you’ll use a for each or a while block. There are also blocks for controlling which screen appears in your app.

Built-in Blocks: Colors

The Colors folder is filled with a set of colors you can use as values for BackgroundColor and other properties. You can also use the make color-color generator to create a specific color using RGB (red, green,blue) values.

Built-in Blocks: Lists

The Lists folder provides blocks for creating lists, adding items to lists, accessing list items, and finding the length of a list.

Built-in Blocks: Variables

Variables are named memory cells-- spreadsheet cells that you name and place numbers, texts, or lists into. For example, if you are calculating the amount of tips earned you might initialize a variable named “tips” and start it out at zero. Once initialized, you can “set” a variable’s value or “get” its current value.

Global Variables can be used in all of your event handlers. Local Variables are event-specific so they only work within a single event handler.

Built-in Blocks: Procedures

With procedures, you can name a set of blocks or instructions. In the example below, there is a procedure “Reset” which is called when the user chooses to start a new game. In this procedure, the score is set back to 0 and the characters are placed back into their starting positions. Once this procedure is defined, you can call it from different places in your app (e.g, when a game ends or when the player explicitly starts a new game).

Built-in Blocks: Logic

The Logic blocks are used in conjunction with if and while blocks to help your app ask complex boolean (true/false) questions. The true and false blocks represent the possible boolean values. the and, or, and not blocks allow you to define a complex condition, and the = block allows you to compare things. Once you drag = in you can use the upside-down triangle to choose a different comparator (<, >, etc.)

Changing Boolean and Height/Width Properties

Boolean properties have 2 options, “true” and “false,” which can be found in the Logic folder of the Built-in Blocks. For example, if you want a Checkbox to be checked set Checkbox.Checked to true.

Height and Width properties are typically set to automatic by default. Automatic means that the height or width of the component will change based on the size of the components inside of it. For example, a HorizontalArrangement component will expand as needed to fit the buttons placed in it.



If you want the width or the height to be the entire length of the screen, then set the property to “fill parent”. Once set to “fill parent”, one can change the AlignHorizontal property, kind of like word, to “Center,” “Left” or “Right” and the AlignVertical property to “Top,” “Center,” or “Bottom.”

Mutators:Adding Slots to Blocks

There are a few blocks that provide mutators which allow you to modify the number of slots in the block. One such block is the + block. When you drag it in, it only has two slots. You can add more slots by clicking on the blue gear icon:

Just click on the gear icon and drag over the “number” blocks into the right side to add more slots.

Mutators can be found in a number of blocks, including make a list which allows you to make a list of any number of items, and in procedure blocks, for specifying how many parameters the procedure should have.
