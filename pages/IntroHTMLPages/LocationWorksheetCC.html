Location: Conceptualize and Customize

Printable Version

Topics

Location component, GPS, URLs

Worksheet Questions

1. What does GPS stand for, who developed it and where does the data come from?

2. What is latitude? What is longitude? Name a place with 0 latitude and one with 0 longitude.

3. In an App Inventor app, when is the LocationSensor.LocationChanged event triggered. If it doesn’t trigger, what might be the problem?

4.  Are GPS satellites the only place from which location information comes?

5. Click the following URLs and figure out what they display:

http://maps.google.com/maps?saddr=37.776938, -122.449555&daddr=37.776523, -122.454490

https://maps.google.com/maps?q=37.77679,-122.4511

6. In a URL like the one above, what does “?” mean? What does “&” mean? How many parameters do the URLs above have?

7. Determine a Maps URL, like the first one above, that shows a map from your hometown to your dorm/apt. in SF.

8. Sketch the blocks for an app that periodically sends a text containing the device’s current address (street address) to some phone number. The app should send the text when it gets a new GPS reading, but at most should send it every minute (hint: check out the properties of LocationSensor to restrict the number of readings).

9. Sketch out the blocks for an app that uses the WebViewer and Google Maps to show directions from the device’s current location to your home.

Customization Exercises

1. Add location information to the No Text While Driving app, i.e., when an incoming text comes in, send the custom reply message but add the phone’s current address to the reply.

2. Create a Breadcrumb app that tracks your whereabouts by recording each location change in a list. You should only record a new breadcrumb if the location has changed by a certain amount, or a certain amount of time has elapsed, because even slight movement can generate a new location reading. You'll need to store the recorded locations in a list, and you should provide some way for the user to view the list.
