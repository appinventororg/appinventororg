Getting a Mole to Appear

To begin, create a new project and drag in a Canvas (under Drawing and Animation Drawer). The Canvas component always comes in way too small, so let's change its width property to "Fill Parent" and its height property to 300 pixels or another fixed amount that would allow the Canvas to take up most of the screen's space.

Now that we have the place where our Mole is supposed to appear and disappear, let's get a mole on the Canvas. The component you should use for the mole is the Image Sprite component (also found in the Drawing and Animation Drawer).

If you test your app after dragging in the Image Sprite, you will not see an Image Sprite because there is no image for your mole. Upload a (small) image into your app and set your ImageSprite1 picture to the image in the properties in the designer. If the picture is too big, either resize it using your computer software, or change the ImageSprite size properties (width and height) to be a small square (ex width-30; height-30).

Getting a Mole to Appear in Random Places on the Screen

In the Designer, drag in a Clock component under the Sensors Drawer and let's take a look at it’s properties.


TimerAlwaysFires means that the Clock is on a loop. A loop means that once it counts down from 1000 ms and gets to 0, it will begin at 1000 ms and begin to count down again.

TimerEnabled means that the timer will count down in milliseconds.

TimerInterval is the number, in Miliseconds, that will be counted down from.

In the blocks editor, the Clock1.Timer eventhandler will be executed each time the TimerInterval reaches 0.

Countdown timer

A countdown timer is simply a label that decreases by one every second and it ends a game when it reaches 0. This is similar to how you would gain a point except it will subtract instead of add and would occur at an event that repeats in a loop.

First we need the label that will show the user how much time is left.


Notice this is being done similarly to how the score was incremented, where there are two labels, one that labels the value of how much time is left with the word "Time" and another label displaying "30" that has the actual amount of time that is left in the game. The components were also renamed so that when one is coding, they know which label should have 1 subtracted from it.
