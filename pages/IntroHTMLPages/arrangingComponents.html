Arranging Components

One of the challenges with App Inventor is designing the screen to look the way you want. Here, we’ll describe how to use special arrangement components to organize your screens.

Aligning the Components in a Screen

By default, a screens AlignHorizontal property is set to “Left” and AlignVertical is set to “Top.” So when you drag in a component, like a button, it goes into the top-left corner. When you add a second component, it appears right below the first..


Often, you’ll want your components to appear centered on the screen. This can be quickly fixed by setting the AlignHorizontal property to "Center" for the Screen1 component.

Arranging Components Side-by-Side


You may also want to layout components side-by-side. To do this, open the Layout drawer in the Palette and drag in a HorizontalArrangement component. The component will appear as a large, empty square within the viewer.



You can drag components (e.g., button, label) into this square so that they appear side-by-side in a horizontal row.

In the figure above, a single button has been dragged into a HorizontalArrangement (the dark outlined box around "Text for button1"). Although it looks like the HorizontalArrangement is full, you can drag other components inside the box and it will expand.

When you add additional components into the arrangement, the trick is to put the Cursor at the right- (or left-) edge. When you release, the component will appear side-by-side next to the other components.

Centering a Horizontal Arrangement

Suppose you want the contents of an arrangement to appear centered in the screen. You first change the HorizontalArrangement’s properties, changing its Width from “Automatic,” meaning it will change its size based on its components, to “Fill parent,” which means it will span the entire length of the phone’s screen.


The buttons will stay on the left side, but if you then change HorizontalArrangement’s AlignHorizontal property to “Center”, the buttons move to the center.

Buttons On Opposite Ends

If you wanted the two components on opposite ends you can set their Width to “Fill parent” and then set the first one’s AlignHorizontal to “Left” and the other’s AlignHorizontal to “Right.


Vertical Arrangements

VerticalArrangements are used less commonly than HorizontalArrangements as objects are placed below each other vertically by default. One reason they are used is to organize areas of the screen that will appear and disappear at certain times. If you know you need to hide a set of components, it’s easier to hide a single arrangement as opposed to hiding each component separately.

Table Arrangements

The TableArrangement is very useful for displaying tabular information. One can put label components into each table cell and name each label appropriately in order to place information in an organized table-like fashion.
