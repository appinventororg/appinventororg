Lesson Plan 4.1:
Using an Index to Step Through a List

Printable Version

Objectives

  • Learn how to iterate through a list using an index and select list item
  • Learn how to use an if conditional to control the iteration.
  • Learn to code using abstract references (e.g., length of list)

Expected Prior Knowledge:

  • Understand the mechanics of building an app.
  • Experience finding and uploading media into App Inventor
  • A basic understanding of variables. http://www.appinventor.org/Variables
  • A basic understanding of how to increment a variable.

Preparation

1. Pre-Task Worksheet: Variables and Incrementing

Handout the pre-task worksheet on variables and incrementing. This can serve as a review of fundamental concepts they’ll need prior to learning about indexes and list iteration.

https://docs.google.com/document/d/1hGTy3hRJG3CuVUBJY9W0w7taFiXma_qm6zoNIfN8aSY

2. Teacher: Go over Objectives/Motivation (5 minutes)

  • Learn how to build a slideshow app. Motivate by emphasizing that the concepts learned can be used to build a quiz app, a study guide, a “Meet your Classmates” app, basically any app in which user navigates through information.
  • Learn a fundamental concept: how to iterate through a list with an index variable

3. Teacher: Demo (10 minutes)

Begin with this starter app (http://appinventor.org/starterApps/slideShowStarter.aia), which has three images loaded and a list variable with the image names.

  • Demo definition of index variable, explain that it tracks the position in list. Draw a memory cell on the board with a 1 in it.
  • Demo Screen.Initialize showing the first image. Explain that you could set the first picture in the designer. Key blocks are select list item which grabs an item from a list, and set Image.Picture, which takes a file name as a parameter. That file name must match a loaded media file exactly.
  • Demo NextButton.Click incrementing index and showing next picture. Show how index memory cell changes to 2 (then 3…). Emphasize how select list item works.
  • Run the app to show how the program bombs out when go past last image.
  • Trace the code on whiteboard, showing how index and Image.Picture change each time NextButton is clicked.

4. Class discussion: how to fix the index out of range error? (10 minutes)

Break the students into small groups and have them discuss how they would fix the problem. Then come together as a class and discuss. The basic solution is:

  • if-statement to check the size of index. Set index to 1 if it is too large.
  • use length of list instead of a fixed number (e.g., 3), so will work even if list is changed.

5. Students: Programming Exercise

Assign the students the task of actually programming the fixes: https://docs.google.com/document/d/1KhtOZLB-2b-PeRNCeMwwBLNI81Asa8uTaCpqO-31w20. This assignment also has a fun exercise of transforming the quiz into a “Meet your Classmates” app.

6. Summarize (expected outcomes, 10 minutes)

Discuss and demonstrate the following, emphasizing the general importance:

  • understand how to use an index variable to track the position in a list.
  • understand how to use if-conditions to check start and end of list.
  • understand that list traversal code should work even if list or list size is changed
