Broadcast Hub: Build, Conceptualize, Customize


Build It

  1. Open a browser and navigate to http://beta.appinventor.mit.edu. Login with your Google account.
  2. Now build the Broadcast Hub app. Follow the instructions in App Inventor chapter 8 using either the paperback version or the pdf version.

Conceptualize It

After completing the Broadcast Hub tutorial, discuss these questions with a partner or small group, jotting down notes while you discuss. Later, create a portfolio page with answers to the questions in your own words.

  1. The NoteTaker app adds data to a list when a submit button is clicked. When is data added to the list with the Broadcast Hub app?
  2. Sketch out, in English, the logic used to determine what to do with each incoming text (it is a nested if-else in the blocks).

Customize It

  1. The app broadcasts each message to everyone, including the phone that sent the message. Modify this so that the message is broadcast to everyone but the sender.
  2. Allow client phones to remove themselves from the list by texting quitabc to the app. You will need a remove from list block.
  3. Let the hub administrator add and remove numbers from the broadcast list through the user interface.
  4. Let the hub administrator specify numbers that should not be allowed into the list.
  5. Customize the app so that anyone can join to receive messages, but only the administrator can broadcast messages.
  6. Customize the app so that anyone can join to receive messages, but only a fixed list of phone numbers can broadcast messages to the group (this is how the Helsinki event app worked; see http://appinventorblog.com/2010/08/25/ success-story-from-helsinki/).
  7. The app stores the broadcast list persistently, but not the log. Each time you close the app and reopen it, the log starts over. Change this so that the log is persistent.
