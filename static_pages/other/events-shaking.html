The user clicks a button, the app responds. A text comes in, the app sends an auto-reply. A flying saucer hits an asteroid, it blows up. Your job as a programmer is to program such event-responses. In App Inventor, you do this directly-- your code is essentially a set of event-responses, of when-do blocks. Check out the following samples:{% endblock %} {% block question %} Example 2. How do you say "I’m shaking” when the user shakes the phone? {% endblock %} {% block answer %}

This app has two components, an AccelerometerSensor and a TextToSpeech. When the user shakes the phone or tablet, the AccelerometerSensor senses the event and the when AccelerometerSensor1.Shaking event handler is triggered. The TextToSpeech component is called to carry out the response of speaking the words “I’m shaking”.
